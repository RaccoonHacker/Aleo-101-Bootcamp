## Private State（私有状态）

### Records

在 Aleo 中，`record` 数据类型用于编码 private state（私有状态）。
Records 的声明方式如下：

```aleo
record Token {
    // The token owner.
    owner: address,
    // The token amount.
    amount: u64,
}
```

Record 的名称不能包含关键字 `aleo`，也不能是其他 record name 的前缀。

Record 内部包含若干 component declarations（组件声明），格式为：

```leo
{visibility} {name}: {type},
```

其中，record component 的名称同样不能包含关键字 `aleo`。

可使用的 visibility qualifier（可见性限定符）包括：

- `constant`
- `public`
- `private`

如果没有显式声明 visibility，Leo 默认使用 `private`。

Record 数据结构必须始终包含一个名为 `owner`、类型为 `address` 的 component。
此外，当把 record 作为 program function 的输入时，还需要 `_nonce: group` 和 `_version: u8` 这两个 component。不过，它们不需要在 Leo program 中手动声明，因为 compiler 会自动插入。

## Public State（公共状态）

### Mappings

Mappings 用于表示 public state（公共状态）中的键值映射。
下面的示例都会基于这个 mapping：

```leo
mapping balance: address => u64;
```

### Querying（查询）

如果只是检查某个 `address` 是否已经在 `balance` 中设置了值，可以使用：

```leo
balance.contains(addr)
Mapping::contains(balance, addr); // Alternate syntax
```

如果要查询某个 `address` 在 `balance` 中对应的值，可以使用：

```leo
balance.get(addr)
Mapping::get(balance, addr); // Alternate syntax
```

需要注意的是，如果 `addr` 对应的值不存在，程序会执行失败。
如果希望在值不存在时使用 fallback value（备用值），可以使用：

```leo
balance.get_or_use(addr,fallback_value)
Mapping::get_or_use(balance, addr, fallback_value); // Alternate syntax
```

程序也可以查询其他 program 中的 mappings，例如：

```leo
let balance1 = credits.aleo::account.get(addr);
let balance2 = credits.aleo::account.get_or_use(addr, 0u64);
```

不过，虽然可以查询其他 program 的 mappings，但一个 program 不能直接修改另一个 program 的 mappings。

### Modifying（修改）

如果要为 `balance` 中的某个 `address` 设置值，可以使用：

```leo
balance.set(addr,value)
Mapping::set(balance, addr, value); // Alternate syntax
```

如果要移除 `balance` 中某个 `address` 已设置的值，可以使用：

```leo
balance.remove(addr)
Mapping::remove(balance, addr); // Alternate syntax
```

### Usage（使用示例）

```leo
program map.aleo {
    mapping balance: address => u64;

    fn dubble() -> Final {
        let addr: address = self.caller;
        return final {
            let current_value: u64 = balance.get_or_use(addr, 0u64);
            balance.set(addr, current_value + 1u64);

            let next_current_value: u64 = balance.get(addr);
            balance.set(addr, next_current_value + 1u64);
        };
    }
}
```

需要注意的是，mapping operations 只能在 `final { }` block 或 `final fn` 中使用。

## Storage Variables（存储变量）

Storage variables 的行为类似于 option types。
它们用于表示 singleton storage variables（单例存储变量），并提供若干查询和修改方法。

下面的示例基于这个 storage variable：

```leo
storage counter: u64;
```

### Querying（查询）

如果要查询 `counter` 当前存储的值，可以使用：

```leo
counter.unwrap();
```

如果 `counter` 还没有被初始化，调用 `unwrap()` 会导致程序执行失败。
如果希望在未初始化时使用 fallback value，可以使用：

```leo
counter.unwrap_or(fallback_value);
```

### Modifying（修改）

如果要为 `counter` 设置值，可以写成：

```leo
counter = 5u64;
```

如果要取消设置 `counter` 的值，可以写成：

```leo
counter = none;
```

### Usage（使用示例）

```leo
program storage_variable.aleo {
    storage counter: u64;

    fn increment() -> Final {
        return final {
            let current_value: u64 = counter.unwrap_or(0u64);
            counter = current_value + 1u64;
        };
    }
}
```

需要注意的是，storage variable operations 只能在 `final { }` block 或 `final fn` 中使用。

### External Access（外部访问）

另一个 program 中定义的 storage variables，可以通过 fully qualified form（完全限定形式）访问：

```leo
program_name.aleo::storage_name
```

External storage variables 是 **read-only（只读）** 的，不能被修改。

例如，假设另一个 program 定义了如下 storage variable：

```leo
program external_program.aleo {
    storage counter: u64;

    ...
}
```

可以在自己的 program 中这样查询它的值：

```leo
let value: u64 = external_program.aleo::counter.unwrap();
```

和 local storage variables 一样，如果该 storage variable 尚未初始化，调用 `unwrap()` 会导致执行失败。
如果希望安全查询并提供 fallback value，可以使用：

```leo
let value: u64 = external_program.aleo::counter.unwrap_or(0u64);
```

External storage variables 不能被赋值，也不能被 unset。下面的写法都是无效的：

```leo
external_program.aleo::counter = 5u64;   // invalid
external_program.aleo::counter = none;   // invalid
```

## Storage Vectors（存储向量）

Storage vectors 的行为类似于 dynamic arrays（动态数组），用于存储某种给定类型的一组值。
它们也提供查询和修改方法。

下面的示例基于这个 storage vector：

```leo
storage id_numbers: [u64];
```

### Querying（查询）

如果要查询 `id_numbers` 中 index 为 `idx` 的元素，可以使用：

```leo
id_numbers.get(idx);
```

它会返回 `u64?`。
如果 `idx` 越界，结果会是 `none`。

如果要获取 `id_numbers` 的当前长度，可以使用：

```leo
id_numbers.len();
```

这个方法始终返回 `u32`，并且不会失败。

### Modifying（修改）

如果要在 index 为 `idx` 的位置设置元素，可以使用：

```leo
id_numbers.set(idx, value);
```

如果要向 `id_numbers` 末尾添加一个元素，可以使用：

```leo
id_numbers.push(value);
```

如果要弹出并返回 `id_numbers` 的最后一个元素，可以使用：

```leo
id_numbers.pop();
```

如果要移除 index 为 `idx` 的元素，并用最后一个元素替换它，可以使用：

```leo
id_numbers.swap_remove(idx);
```

如果要清空 `id_numbers` 中的所有元素，可以使用：

```leo
id_numbers.clear();
```

需要注意的是：

- `clear()` 并不会真正删除 vector 中的任何值，它只是把 length 设置为 `0`。
- 类似地，`swap_remove()` 和 `pop()` 也不会物理删除值。它们只是把 length 减少 `1`，从而确保最后一个元素不再可访问。

### Usage（使用示例）

```leo
program storage_vector.aleo {
    storage id_numbers: [u64];

    fn add_id(new_id: u64) -> Final {
        return final {
            id_numbers.push(new_id);
        };
    }

    fn remove_id(idx: u32) -> Final {
        return final {
            id_numbers.swap_remove(idx);
        };
    }
}
```

需要注意的是，storage vector operations 只能在 `final { }` block 或 `final fn` 中使用。

### External Access（外部访问）

另一个 program 中定义的 storage vectors，可以通过 fully qualified form 访问：

```leo
program_name.aleo::storage_name
```

External storage vectors 是 **read-only（只读）** 的，不能被修改。

例如，假设另一个 program 定义了如下 storage vector：

```leo
program external_program.aleo {
    storage id_numbers: [u64];
}
```

可以在自己的 program 中查询其中的元素或长度：

```leo
let first: u64? = external_program.aleo::id_numbers.get(0u32);
let length: u32 = external_program.aleo::id_numbers.len();
```

External storage vectors 不能被修改。下面这些操作都是无效的：

```leo
external_program.aleo::id_numbers.push(5u64);        // invalid
external_program.aleo::id_numbers.set(0u32, 5u64);   // invalid
external_program.aleo::id_numbers.pop();             // invalid
external_program.aleo::id_numbers.swap_remove(0u32); // invalid
external_program.aleo::id_numbers.clear();           // invalid
```

## 小结

这一节主要说明 Leo 中的两类状态：

第一类是 **private state**，主要通过 `record` 表示。Record 用于承载私有数据，必须包含 `owner: address`，并且默认字段可见性为 `private`。

第二类是 **public state**，主要包括：

- `mapping`
- `storage variable`
- `storage vector`

这些公共状态可以被查询和修改，但修改操作通常只能放在 `final { }` block 或 `final fn` 中执行。
如果访问的是其他 program 中的 mapping、storage variable 或 storage vector，则一般只能读取，不能直接修改。
